# Тестовое задание Frontend разработчик на React.

Вам дан проект Дашборда со списком звонков, для операторов, в котором они могут просматривать детали звонка и менять их статусы.

```
Проект уже работает, но в нем намеренно допущены ошибки, которые вам предстоит выявить и исправить.
```

## Что нужно сделать:

- Найти и исправить ошибки в коде
- Правильно обработать ошибки от API
- Улучшить качество и читаемость кода

## На что стоит обратить внимание:

- Что может вызывать проблемы с производительностью
- Работа с React Query и обработка ошибок
- Работа с состоянием
- Типизация

## Чего делать не нужно:

- Полностью рефакторить код
- Тратить время на стилизацию

## Как отправить свое задание на проверку:

- Форкнуть репозиторий на GitHub
- Внести изменения в master ветке
- Описать в README какие баги и недочеты вы обнаружили, исправили и почему
- Отправить ссылку на репозиторий в GitHub

## Фиксы ##

1. В calls.ts функция принимала 2 параметра id, calls а для mutationFn: updateCallStatus в хуке useCalls.ts нужен 1 объект.
2. в socket.ts:
2.1. при повторном вызове connectSocket создавался новый интервал, но старые не удалялись, что приводит к множественным интервалам.
2.2. при размонтировании компонента не удалялся listener.
2.3. добавлена функция disconnectSocket для очистки интервала
3. useSocket.ts 
3.1. хук useSocket был реализован, но не был подключен, обновление статуса через WebSocket не работали. Подключил в Dashboard.tsx
3.2. тип update: Partial<Call>, но его приводили к типу Call. Данные перетирались, а не мержились, что приводило к потере поля `phone`, `duration`.
3.3. добавлено отключение от сокета (очистка интервала) при размонтировании
3.4. нет зависимости от queryClient в хуке. 
4. useCalls.ts
4.1. излишне refetchOnWindowFocus: true так как при фокусе на окно каждый раз выполняется запрос к API, но данные обновляюсят по WS. 
4.2 изменил типы, избавился от использования any
4.3. добавлен cancelQueries, без которого фоновые запросы могут перезаписать оптимистичное обновление.
4.4. добавил onSuccess для корректного обновления только после успешного запроса и отображения лоадера 'updating...'
4.5. без onSettled нет синхронизации с сервером после завершения мутации. 
5. CallsList.tsx
5.1. добавил виртуализацию рендера списка звонков через react-virtual
5.2. рендер текста ошибки 
5.3. защита от отсутсвия data, чтоб не использовать data!, т.к. если запрос упадет data будет undefined
6. uiStore.tsx
    в сторе храним только id звонка, чтоб иметь актуальные данные при обновлении по WS или мутации. При рендере получаем текущие данные из callsQuery. 
7. CallDetails.tsx
7.1 очистка ошибки при переключении на другой звонок
7.2. лоадер при выполнении обновления + отключение кнопки при выполнении и если статус установлен в hold.
7.3. отображение ошибки.
